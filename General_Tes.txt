Menurut dari yang saya baca, JSON Web Token (JWT) itu salah satu standar JSON (Javascript Object Notation) untuk keperluan akses token. Token dibentuk dari kombinasi beberapa informasi yang di-encode dan di-enkripsi. Kombinasi beberapa informasinya adalah header, payload dan signature.

- Header, isinya itu jenis algoritma yang dipake untuk generate signature.
- Payload, isinya itu data penting untuk keperluan otentikasi. Data ini dalam konteks JWT biasa disebut CLAIMS.
- Signature, isinya itu hasil dari enkripsi data menggunakan algoritma kriptografi. Datanya gabungan dari (encoded) header, (encoded) payload, dan secret key.

JWT biasanya dipakai untuk menerapkan teknik otorisasi menggunakan token, token di-generate di back end secara acak (menggunakan algoritma tertentu) terus disimpan di database bersamaan dengan data user. Token tersebut bisa jadi tidak ada isinya, cuma random string, atau bisa juga ada isinya.

Nantinya di setiap http call, token yang disertakan di request header dicocokan dengan token yang ada di database, terus dilanjutin ke pengecekan, untuk ditentukan apakah request tersebut adalah verified request yang memang berhak mengakses endpoint.
